A torch in the dark

The engine existed. The last mile didn't.

osxEQL pairs open-source Wine (from CrossOver's LGPL source) with DXMT — DirectX 11 translated straight to Metal — so EverQuest Legends runs on Apple Silicon with zero proprietary layers. It's brilliant at playing the game.

But a clean, first-time install couldn't get past the front door: the launcher crashed before you could even log in. It turned out to be three specific walls — and they're all gone now.

What we broke through

Three walls, cleared.

Wall 01 · Runtime

The launcher had no GPU or fonts

osxEQL's Wine shipped without libvulkan.1.dylib (MoltenVK) and an x86-64 FreeType — but the launcher is a Chromium/CEF app that needs Vulkan, and Wine needs FreeType. We drop the right x86-64 libraries into the Wine tree where dlopen actually looks.

Wall 02 · Services

No RPC subsystem

The minimal prefix never started rpcss, so every COM/RPC call died with RPC_S_SERVER_UNAVAILABLE. We bring it up before launch.

Wall 03 · The big one

A bootstrapper that eats itself

The first-run LaunchPad.exe loads Daybreak's npdg download plugin, which crashes under Wine. But it first downloads the real CEF launcher into a mangled folder literally named C:. Rename it, run the real launcher directly — and you're at the login screen.

Quickstart

Three commands to Norrath.

You bring an Apple Silicon Mac, a Daybreak/EQ Legends account, and the official EQLegends_setup.exe. The tool does the rest.

# 1 · get the tools
git clone https://github.com/jkatsnelson/osxeql-qol
cd osxeql-qol

# 2 · one command: adds MoltenVK+FreeType, installs, bypasses the bootstrapper, opens the login
./eql-mac.sh setup ~/Downloads/EQLegends_setup.exe

# 3 · log in, let it pull the ~6.9 GB client, hit Play
./eql-mac.sh play        # eqgame → DXMT → Metal
# CRender::InitDevice completed successfully.
→The launcher opens the real Game Jawn / Daybreak login. Sign in with your account.
→It downloads the ~6.9 GB classic client (pre-Kunark). Keep your Mac awake — the tool handles that.
→Hit Play. eqgame renders through DXMT/Metal, windowed, mouse mapped 1:1.

Community maps

Fixing "it's so dark"

The old gamma slider is dead under Wine. EQ Legends' DX11 client has an in-engine Enhanced Vision slider (Options → Display → Advanced) that works — plus the honest truth about race night-vision and torches.

No double cursor

StealthPointer hides the macOS pointer over the game — F1 / F2.

DXMT tuning

A sane dxmt.conf: frame pacing, Metal 3.2 shaders, alt-tab handling.

doctor.sh

One read-only check for Rosetta, the libs, the prefix, disk, and the DXMT symbol.

Windowed & mapped
